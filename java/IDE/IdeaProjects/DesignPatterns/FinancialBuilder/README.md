# Financial Report Builder

A Java implementation of the **Builder Design Pattern** for generating financial reports in different formats.

## ğŸ“‹ Project Overview

This project demonstrates the Builder pattern through a financial reporting system that can generate reports in various formats (currently PDF, easily extensible to HTML, Excel, etc.). The pattern separates the construction logic from the representation, making it easy to add new report formats without modifying existing code.

## ğŸ—ï¸ Design Pattern: Builder

The Builder pattern is used here to construct complex [`Report`](src/main/java/org/elainehello/finance/api/Report.java) objects step by step. It's particularly useful when:

- Creating complex objects with many optional parameters
- The construction process must allow different representations
- You want to isolate the construction code from the business logic

### Pattern Components

- **Product**: [`Report`](src/main/java/org/elainehello/finance/api/Report.java) - The complex object being built
- **Builder**: [`ReportBuilder`](src/main/java/org/elainehello/finance/api/ReportBuilder.java) - Abstract interface defining construction steps
- **Concrete Builder**: [`PdfReportBuilder`](src/main/java/org/elainehello/finance/builders/PdfReportBuilder.java) - Implements specific format construction
- **Director**: [`ReportDirector`](src/main/java/org/elainehello/finance/core/ReportDirector.java) - Controls the construction sequence
- **Client**: [`Main`](src/main/java/org/elainehello/finance/app/Main.java) - Uses the pattern to create reports

## ğŸ’¡ Usage Example

```java
// 1. Prepare your data
List<Transaction> transactions = Arrays.asList(
    new Transaction(LocalDate.now(), "Cloud Server Fee", new BigDecimal("150.00")),
    new Transaction(LocalDate.now(), "License Renewal", new BigDecimal("49.99"))
);

// 2. Setup the builder pattern
ReportDirector director = new ReportDirector();
ReportBuilder builder = new PdfReportBuilder();

// 3. Generate the report
Report report = director.construct(builder, "TechCorp LLC", transactions);
report.display();
```

## ğŸ”§ Extending the System

### Adding New Report Formats

To add a new report format (e.g., HTML), create a new concrete builder:

```java
public class HtmlReportBuilder extends ReportBuilder {

    @Override
    public void buildHeader(String companyName) {
        report.setFormat("HTML");
        report.addSection("<h1>Financial Report for " + companyName + "</h1>");
    }

    @Override
    public void buildContent(List<Transaction> data) {
        StringBuilder table = new StringBuilder("<table>");
        table.append("<tr><th>Date</th><th>Description</th><th>Amount</th></tr>");
        for (Transaction t : data) {
            table.append(String.format(
                "<tr><td>%s</td><td>%s</td><td>%s</td></tr>",
                t.date(), t.description(), t.amount()
            ));
        }
        table.append("</table>");
        report.addSection(table.toString());
    }

    @Override
    public void buildFooter() {
        report.addSection("<footer>Generated by FinanceService v1.0</footer>");
    }
}
```

### Adding New Data Types

The [`Transaction`](src/main/java/org/elainehello/finance/api/Transaction.java) record can be extended or new data types can be created following the same pattern.
